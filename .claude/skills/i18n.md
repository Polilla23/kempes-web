# Skill: Internationalization Helper

Agrega traducciones EN/ES cuando se crean nuevas features o se agregan textos.

## Uso

```
/i18n [archivo o componente]
```

Ejemplos:
- `/i18n frontend/src/routes/management/trophies/` - Agregar traducciones para nueva página
- `/i18n` - Buscar textos hardcodeados en archivos recientes

## Instrucciones

Cuando el usuario ejecute este skill, seguir estos pasos:

### 1. Identificar textos a traducir

Buscar en los archivos especificados:
- Strings hardcodeados en JSX/TSX
- Mensajes de toast/notificaciones
- Labels de formularios
- Textos de botones
- Mensajes de error
- Títulos y descripciones
- Placeholders

**Patrones a buscar:**
```tsx
// ❌ Textos hardcodeados
<h1>Clubs Management</h1>
<Button>Create New</Button>
toast.success("Club created!")
placeholder="Enter name"

// ✅ Con i18n
<h1>{t('clubs:title')}</h1>
<Button>{t('clubs:create.button')}</Button>
toast.success(t('clubs:create.success'))
placeholder={t('clubs:placeholders.name')}
```

### 2. Determinar el namespace

Elegir namespace basado en la feature:
- Páginas de gestión: `[feature]` (clubs, players, etc.)
- Navegación: `navigation`
- Autenticación: `auth`
- Componentes comunes: `common`

### 3. Generar estructura de claves

Seguir la estructura estándar del proyecto:

```json
{
  "title": "Feature Title",
  "create": {
    "title": "Create New Entity",
    "description": "Add a new entity to the system",
    "button": "New Entity",
    "action": "Create",
    "success": "Entity created successfully!",
    "error": "Error creating entity"
  },
  "edit": {
    "title": "Edit Entity",
    "description": "Make changes to the entity",
    "button": "Update Entity",
    "action": "Edit",
    "success": "Entity updated successfully!",
    "error": "Error updating entity"
  },
  "delete": {
    "confirm": "Are you sure you want to delete this entity?",
    "action": "Delete",
    "success": "Entity deleted successfully!",
    "error": "Error deleting entity"
  },
  "fields": {
    "name": "Name",
    "description": "Description"
  },
  "labels": {
    "entityName": "Entity Name",
    "entityDescription": "Entity Description"
  },
  "placeholders": {
    "entityName": "Enter entity name",
    "entityDescription": "Enter description (optional)"
  },
  "buttons": {
    "create": "Create Entity",
    "update": "Save Changes",
    "cancel": "Cancel"
  },
  "table": {
    "search": "Search",
    "actions": "Actions",
    "noResults": "No entities found"
  }
}
```

### 4. Crear archivos de traducción

#### Inglés
`frontend/src/i18n/locales/en/[feature].json`

#### Español
`frontend/src/i18n/locales/es/[feature].json`

**Importante**: Mantener las mismas claves en ambos archivos.

### 5. Registrar en configuración i18n

Archivo: `frontend/src/i18n/config.ts`

```typescript
// Agregar imports
import enFeature from './locales/en/[feature].json'
import esFeature from './locales/es/[feature].json'

// Agregar a resources
resources: {
  en: {
    // ... existentes
    feature: enFeature,
  },
  es: {
    // ... existentes
    feature: esFeature,
  },
},

// Agregar a namespaces
ns: [
  // ... existentes
  'feature',
],
```

### 6. Actualizar componentes

Reemplazar textos hardcodeados:

```tsx
import { useTranslation } from 'react-i18next'

function Component() {
  const { t } = useTranslation('feature')  // namespace

  return (
    <div>
      <h1>{t('title')}</h1>
      <Button>{t('create.button')}</Button>
    </div>
  )
}
```

### 7. Verificación

Indicar al usuario que verifique:

1. Cambiar idioma en la app (language switcher)
2. Verificar que todos los textos cambian correctamente
3. Revisar la consola por warnings de claves faltantes

## Traducciones comunes

### Acciones
| Inglés | Español |
|--------|---------|
| Create | Crear |
| Edit | Editar |

| Save | Guardar |

## Namespaces existentes

Revisar en `frontend/src/i18n/config.ts`:
- common, navigation, auth
- users, clubs, players
- eventTypes, competitionTypes
- seasons, salaryRates, fixtures

## Errores comunes

### Clave no encontrada
Si ves warnings como `i18next: key "feature:key" not found`:
1. Verificar que el archivo JSON existe
2. Verificar que está importado en config.ts
3. Verificar que el namespace está en el array `ns`
4. Verificar la escritura correcta de la clave

### Interpolación
Para valores dinámicos:
```json
{
  "welcome": "Welcome, {{name}}!"
}
```
```tsx
t('welcome', { name: 'John' })
```
